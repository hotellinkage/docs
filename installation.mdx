# Installation

Get your booking engine running in under 10 minutes. Just add 2 files to your website and configure your hotel ID.

<Note>
**What you need:** Your Hotel Linkage property ID | **Time required:** ~5 minutes
</Note>

## Add 2 Files

<Steps>
  <Step title="CSS in Header">
    Add the stylesheet to your `<head>` section
  </Step>
  
  <Step title="JavaScript in Body">
    Add the script at the end of your `<body>`
  </Step>
  
  <Step title="Configure Hotel">
    Set your hotel ID and preferences
  </Step>
</Steps>

## Main Files

### Stylesheet

The first file is the default theme stylesheet (index.css), which defines CSS variables without interfering with your project's existing styles. Include it in the `<head>` of your page using the preload technique to optimize performance:

```html
<link
  rel="preload"
  as="style"
  href="https://static.mirai.com/core/index.css"
  onload="this.onload=null;this.rel='stylesheet'"
/>
```

### Component Script

The second file is the JavaScript component repository (index.js), which uses an asynchronous loading system to minimize impact on page performance. Add it at the end of the `<body>`:

```html
<script type="module" defer src="https://static.mirai.com/core/index.js" 
data-cookieconsent="necessary"></script>
```

## Data Repository Instance

Specify the data repository you want to load using a `<div>` element in the `<body>` of your page. This element identifies the hotel or chain associated with the core and is essential for initializing the components:

```html
<div data-mirai-id="XXXXXXX"></div>
```

### Configuration Attributes

<CardGroup cols={1}>
  <Card title="data-mirai-id" icon="key">
    **Type:** `string` (required)  
    Unique identifier of the hotel or chain
  </Card>
  
  <Card title="data-type" icon="inbox">
    **Type:** `string` **Default:** `"hotel"`  
    Instance type: "hotel" or "chain"
  </Card>
  
  <Card title="data-hotel" icon="building">
    **Type:** `string` **Default:** None  
    Hotel identifier (only for chains)
  </Card>
  
  <Card title="data-currency" icon="dollar-sign">
    **Type:** `string` **Default:** None  
    3-letter currency code (e.g., EUR, USD, JPY) that defines the currency for all components
  </Card>
  
  <Card title="data-locale" icon="globe">
    **Type:** `string` **Default:** None  
    Language code (e.g., en-GB, es-ES) that sets the language of the components
  </Card>
</CardGroup>

<Note>
The data-locale attribute is subject to priority rules. Its value will only be applied if no lang parameter is specified in the page URL and no lang attribute exists in the `<html>` tag of the document. In that case, the latter take priority.
</Note>

## Specific Components

The files and data repository form the foundation for the engine to function, but to implement a specific component (e.g., `<rates>` for the room listing or `<finder>` for the search), you must include its specific HTML on the page. Refer to the "Components" subsections for the required markup for each one.
